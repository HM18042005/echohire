1. Request & security path

  • • Flutter sends Bearer-authenticated requests to FastAPI.
  • • verify_firebase_token() gates user-facing endpoints.
  • • verify_static_bearer_token() protects /tools/end-interview.
  • • Webhooks optionally verify HMAC signatures with VAPI_WEBHOOK_SECRET.

2. Orchestration path

  • • /workflow/* uses InterviewSetupAssistant as a session-state machine.
  • • It collects role, interview type, and experience level before generating questions.
  • • /interviews/ai-guided and /start-ai bridge backend interview records with Vapi calls.
  • • The backend keeps business state even when Vapi or Gemini falls back to mock/offline behavior.

3. Persistence model

  • • Firestore collections seen in code: profiles, interviews, feedback, transcripts, interview_sessions, ai_guided_sessions.
  • • Interview docs are the central record tying together user, status, Vapi call ID, and scores.
  • • Transcript and feedback are stored separately, then linked back to the interview doc.
  • • Several routes use mock fallbacks if Firestore is disabled or unavailable.

4. AI + voice lifecycle

  • • Vapi runs the live interview/call lifecycle and emits status/transcript metadata.
  • • Gemini handles question generation and transcript-based evaluation.
  • • Transcript retrieval is layered: stored transcript → transcript URL → Vapi polling retries.
  • • Feedback can be generated automatically from webhook completion or on-demand via /ai-feedback.
